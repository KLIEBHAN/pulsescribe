Latest pass - shared permissions card UX (`ui/permissions_card.py`)
- Restflächen-Inventur under `ui/` after excluding Welcome / Settings / Overlay / Menubar / Onboarding focused on:
  - `ui/permissions_card.py`
  - `ui/hotkey_card.py`
  - `ui/styles_windows.py` (style helper only)
  - `ui/animation.py` (not an interactive surface)
- Selected area: `ui/permissions_card.py` because it is still a visible shared status/action surface with broad user impact.
- Main weaknesses found:
  1. Initial permission states were too vague (`…`, empty summary).
  2. The summary area was effectively one line high even though the feedback text is longer.
  3. Some ready states were too generic (`Enabled`) and did not explain what became available.
  4. Permission action buttons lacked contextual guidance.
- Changes in this pass:
  - improved initial checking text and clearer waiting/checking microcopy during refresh
  - made ready states more descriptive (`auto-paste ready`, `Hold hotkeys ready`)
  - enlarged the summary feedback area and enabled multiline wrapping
  - widened action buttons slightly and added contextual tooltips for the permission actions
