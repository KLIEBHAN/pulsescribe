UX notes - 2026-04-19

Latest pass - Welcome + status feedback UX (`ui/welcome.py`, overlays)
- Scope: focused the next high-value UX area on the Welcome window and status/overlay feedback surfaces without reworking the recently touched Windows Settings/Onboarding flows again.
- Reviewed weaknesses first:
  1. Welcome still used weak provider/API-key feedback (`✓` / `✗`) instead of showing what is required vs optional.
  2. The Setup → Try it section was not very helpful when no hotkey was configured and its hotkey display was hard to scan.
  3. Welcome save feedback still relied mostly on the button label and lacked a persistent status hint.
  4. Overlay error/loading/status text was not fully consistent across implementations, and macOS error feedback hid useful detail.
- Changes in this pass:
  - added mode-aware provider guidance text in Welcome
  - replaced Welcome API-key checkmarks with inline states like `Required`, `Configured`, `Optional`, `Not needed`
  - improved API-key placeholders/tooltips in Welcome
  - improved Setup → Try it messaging, including a clearer empty state when no hotkey exists
  - formatted displayed hotkeys into friendlier labels (for example `Option+Space`, `Fn`)
  - made the Setup → Try it card refresh after hotkey changes
  - improved preset feedback copy in Setup and added persistent footer save feedback in Welcome
  - unified overlay feedback text handling across macOS / PySide6 / Tk overlays
  - macOS overlay now shows concise error details instead of only `Error`
