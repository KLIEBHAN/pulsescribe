UX notes - 2026-04-21

Latest pass - secondary settings surfaces (`ui/secondary_settings_feedback.py`, `ui/settings_windows.py`, `ui/welcome.py`)
- Focused on the remaining Refine / output / clipboard settings after reviewing their weak guidance and section-level feedback.
- Main weaknesses found:
  1. Refine, clipboard and display options were technically correct, but not explained in a way that made their trade-offs obvious.
  2. Section-level Save & Apply expectations were mostly implicit, so unsaved changes vs. unchanged states were hard to spot.
  3. Windows and macOS used noticeably different wording for the same secondary settings.
  4. RTF / clipboard restore / refine model override copy leaned too technical or too terse.
  5. The macOS Refine card had weak grouping and cramped hierarchy for several unrelated toggles.
- Changes in this pass:
  - added shared feedback helpers for refine-model guidance, refine section summaries, and output / clipboard summaries
  - improved Windows secondary settings copy, tooltips, accessible names and live section feedback for unchanged vs. unsaved states
  - clarified Windows model guidance so the selected provider's default model is visible without making the field feel mandatory
  - renamed several labels to be less technical, especially around transcript cleanup, RTF, overlay and clipboard restore
  - improved the macOS Refine tab hierarchy with clearer section grouping, better descriptions, provider/model guidance and section status summaries
  - added macOS section feedback for unsaved vs. already-applied settings and clearer relaunch wording for Dock icon changes
