UX notes - 2026-04-19

Latest pass - Shared action/status card UX (`ui/hotkey_card.py`, `ui/permissions_card.py`)
- Scope: improved the remaining broad-impact shared UI cards outside Onboarding / Settings / Welcome / Menubar.
- Reviewed flows:
  - shared Hotkey card
  - shared Permissions card
- Main weaknesses found:
  1. Hotkeys were still shown in raw uppercase forms, which were harder to scan quickly.
  2. The Hotkey empty/recording states were functional but not especially actionable.
  3. Permission row statuses and button labels were sometimes too terse or technical.
  4. The Permissions card lacked a clear card-level summary / “all set” / checking message.
- Changes in this pass:
  - formatted displayed hotkeys into friendlier labels like `Option+Space` and `Fn`
  - improved Hotkey empty-state copy and added clearer recording/cancelled guidance
  - improved shared permission microcopy (`Allow`, `Open Settings`, clearer row statuses)
  - added a card-level permission summary message for blocked / checking / partially ready / all set states
  - widened permission action buttons a bit to fit clearer labels without changing behavior
- Validation:
  - `pytest tests/test_hotkey_card.py tests/test_permissions_card.py tests/test_onboarding.py tests/test_settings_ui.py -q`
  - `python -m py_compile ui/hotkey_card.py ui/permissions_card.py tests/test_hotkey_card.py tests/test_permissions_card.py`
  - `ruff check ui/hotkey_card.py ui/permissions_card.py tests/test_hotkey_card.py tests/test_permissions_card.py`
