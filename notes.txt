UX notes - 2026-04-19

Latest pass - Menubar action flow UX (`ui/menubar.py`)
- Scope: targeted the next broad-impact UI area outside Onboarding / Settings / Welcome by improving the menu bar action menu and its state feedback.
- Reviewed weaknesses first:
  1. The menu itself exposed too little context; it mostly showed a static app title plus vague actions.
  2. Action labels like `Settings...`, `Open Logs`, and `Export Diagnostics…` were understandable for power users but not especially clear or screenreader-friendly.
  3. Opening logs / exporting diagnostics had almost no direct feedback in the UI; failures were effectively silent unless users checked logs.
  4. The menubar title relied too heavily on icons, so short transient states were easy to miss.
- Changes in this pass:
  - improved menubar title feedback for central transient states (`Listening…`, `Recording…`, `Loading…`, etc.)
  - added disabled current-status + hint rows at the top of the menu for better guidance and scanability
  - renamed actions for clarity: `Open Setup & Settings…`, `Open Log File`, `Export Redacted Diagnostics…`, `Quit PulseScribe`
  - added inline menu feedback for explicit actions like opening logs and exporting diagnostics
  - improved failure feedback for diagnostics / log opening without changing the underlying actions
  - kept the existing daemon / menu wiring intact
- Validation:
  - `pytest tests/test_menubar.py tests/test_daemon_mode.py -q`
  - `python -m py_compile ui/menubar.py tests/test_menubar.py`
  - `ruff check ui/menubar.py tests/test_menubar.py`
