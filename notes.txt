Latest pass - daemon / retry / status UX outside the overlay (`ui/daemon_status_feedback.py`, `ui/menubar.py`, `pulsescribe_daemon.py`, `pulsescribe_windows.py`)
- Focused on non-overlay daemon state feedback after reviewing the remaining startup / retry / recovery surfaces.
- Main weaknesses found:
  1. macOS menu bar loading / error states were often too generic to explain what PulseScribe was doing.
  2. Windows tray status ignored same-state text changes, so loading / warmup progress could stay stale.
  3. Recovery guidance was inconsistent or missing, especially around transient errors and automatic return-to-ready behavior.
  4. Non-overlay status copy was duplicated and inconsistent across macOS + Windows.
- Changes in this pass:
  - added shared daemon-status copy helpers for concise non-overlay labels, loading guidance and error recovery hints
  - improved macOS menu bar status + hint text for loading, warmup, missing API keys, busy states and timeout-style failures
  - improved Windows tray titles so they reflect the latest state text even when the state itself does not change
  - aligned Windows tray microcopy with the newer app wording and made status titles more descriptive / accessible
  - started passing clearer friendly error labels from daemon timeout / worker failure paths instead of falling back to generic `Error` everywhere
