Latest pass - shared hotkey card UX (`ui/hotkey_card.py`)
- Re-checked the remaining shared visible card flow and focused on `ui/hotkey_card.py` as the next still-meaningful UX surface.
- Main weaknesses found:
  1. The shared card depended too much on callers to provide save/error feedback.
  2. Some microcopy was slightly mouse-centric or too terse for cancellation / recording states.
  3. Fields and buttons lacked contextual hints for keyboard / screenreader-adjacent guidance.
  4. Longer feedback text could feel cramped in smaller card widths.
- Changes in this pass:
  - added fallback success/error feedback when callers do not set a status themselves
  - improved empty / recording / cancelled microcopy and shortened the recording status text
  - added contextual tooltips to preset buttons, record buttons and hotkey fields
  - refreshed field placeholders / hints so the card reads better in tighter layouts
